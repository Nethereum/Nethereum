# AppChain MUD Example Template

This template demonstrates how to build a MUD-based application on AppChain.

## Project Structure

```
contracts/
├── mud.config.ts              # MUD table definitions
├── .nethereum-gen.multisettings  # Nethereum code generation config
└── src/
    └── systems/
        ├── AdminSystem.sol       # App configuration and permissions
        ├── UserProfileSystem.sol # User profile management
        └── InventorySystem.sol   # User inventory management

Generated/                     # Auto-generated by Nethereum
├── Tables/                    # Table services from mud.config.ts
└── Systems/                   # System services from Solidity
```

## Tables

| Table | Key Type | Description |
|-------|----------|-------------|
| AppConfig | Singleton (no key) | Global app configuration |
| UserProfile | Single key (address) | User profiles |
| UserItem | Composite key (address, uint256) | User inventory items |
| CatalogEntry | Single key (uint256) | Catalog with array fields |
| Permission | Triple key (bytes32, bytes32, address) | RBAC permissions |
| Counter | Single key (bytes32) | ID generation counters |

## Systems

### AdminSystem
- `initialize(appName, version, maxUsers)` - Initialize the app
- `pause()` / `unpause()` - Toggle app pause state
- `grantRole(resource, role, account)` - Grant permissions
- `revokeRole(resource, role, account)` - Revoke permissions
- `transferAdmin(newAdmin)` - Transfer admin rights

### UserProfileSystem
- `createProfile(username, bio)` - Create a new profile
- `updateProfile(username, bio)` - Update profile info
- `deactivateProfile()` / `reactivateProfile()` - Toggle active state

### InventorySystem
- `addItem(itemId, quantity)` - Add items to inventory
- `removeItem(itemId, quantity)` - Remove items from inventory
- `transferItem(to, itemId, quantity)` - Transfer items between users

## Code Generation

### Prerequisites
1. Install MUD: `pnpm install`
2. Compile contracts: `pnpm mud build`
3. Install Nethereum.Generator.Console

### Generate C# Code

```bash
cd contracts

# Generate table services from mud.config.ts
Nethereum.Generator.Console generate from-mud-config \
  --mudConfig mud.config.ts \
  --namespace Nethereum.AppChain.Templates.MudExample.Tables \
  --outputPath ../Generated/Tables

# Generate system services from compiled contracts
Nethereum.Generator.Console generate from-abi \
  --abiFile out/UserProfileSystem.sol/UserProfileSystem.json \
  --namespace Nethereum.AppChain.Templates.MudExample.Systems.UserProfileSystem \
  --outputPath ../Generated/Systems/UserProfileSystem

# Or use multisettings for batch generation
Nethereum.Generator.Console generate from-multisettings \
  --settingsFile .nethereum-gen.multisettings
```

## Usage with AppChain

```csharp
// Initialize AppChain with MUD World
var dataChain = new AppChain(config, blockStore, txStore, receiptStore, logStore, stateStore);
await dataChain.InitializeAsync(new GenesisOptions
{
    DeployMudWorld = true,
    DeployCreate2Factory = true,
    PrefundedAddresses = new[] { sequencerAddress }
});

// Start sequencer
var sequencer = new Sequencer(dataChain, sequencerConfig);
await sequencer.StartAsync();

// Use generated services
var web3 = new Web3(rpcUrl);
var userProfileService = new UserProfileSystemService(web3, worldAddress);

// Create a profile
var tx = await userProfileService.CreateProfileRequestAsync("alice", "Hello AppChain!");
```

## Bootstrap Policy

For development, use the bootstrap policy before deploying to L1/L2:

```csharp
var bootstrapConfig = new BootstrapPolicyConfig
{
    AllowedWriters = new List<string> { sequencerAddress, adminAddress },
    AllowedAdmins = new List<string> { adminAddress },
    OpenWriterAccess = false  // Restrict to allowlist
};

var policyService = new BootstrapPolicyService(bootstrapConfig);
```
