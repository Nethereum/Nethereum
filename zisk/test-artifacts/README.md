## Witness format

Each `_standard.bin` file uses the Zisk standard input format:

```
[8 bytes] total_size = 16 + witness_length
[8 bytes] reserved = 0
[8 bytes] witness_length
[N bytes] witness data (BinaryBlockWitness serialized)
```

The witnesses are generated by the `GenerateTestWitness` test class in `tests/Nethereum.EVM.Core.Tests/GenerateTestWitness.cs`.

## How to reproduce

### Prerequisites

- Zisk v0.17.0 with the 5 NativeAOT constraint fixes (branch `fix/nativeaot-constraints` on `Nethereum/zisk` fork)
- `HWLOC_COMPONENTS=-gl` set on headless servers
- provingKey generated for v0.17.0 (see `zisk/scripts/proving/` for generation scripts)

### Execute (emulator only, no proof)

```bash
export HWLOC_COMPONENTS=-gl
cargo-zisk execute -e evm_lazy_nocookie -i witness_eth_transfer_noroot_standard.bin -l
```

### Verify constraints (no proof, checks all constraint satisfaction)

```bash
cargo-zisk verify-constraints -e evm_lazy_nocookie -i witness_eth_transfer_noroot_standard.bin -l -d
```

### Prove and verify

```bash
cargo-zisk prove -e evm_lazy_nocookie -i witness_eth_transfer_noroot_standard.bin -l -n -o proof_output
cargo-zisk verify -p proof_output
```

### Verify pre-built proofs (no proving key needed)

The `proofs/` directory contains verified STARK proofs (329KB each). To verify them independently:

```bash
cargo-zisk verify -p proofs/proof_eth_transfer_noroot
cargo-zisk verify -p proofs/proof_sstore_noroot
cargo-zisk verify -p proofs/proof_eth_transfer_patricia
cargo-zisk verify -p proofs/proof_sstore_patricia
```

All should output `STARK proof was verified` in ~187ms.

### What each witness tests

- **eth_transfer_noroot**: Simplest provable transaction — sender transfers ETH to receiver, gas=21000, no storage, no state root. Tests basic EVM execution, ECDSA signature recovery, balance transfer.
- **sstore_noroot**: Contract call executing `PUSH 0x42, PUSH 0x00, SSTORE, STOP` — tests EVM storage writes, gas metering (20000 gas for SSTORE), contract code loading.
- **eth_transfer_patricia**: Same as eth_transfer but computes Patricia Merkle trie state root after execution. Tests the full state commitment pipeline.
- **sstore_patricia**: SSTORE + Patricia state root. Tests storage modification + trie update + state commitment.
- **contract_creation**: Deploy a contract with init code (CODECOPY + RETURN) that installs a SSTORE runtime. Tests CREATE opcode, code storage, init code execution.
- **sha256_precompile**: Contract calls SHA256 precompile (address 0x02) via STATICCALL with 32 bytes of input. Tests CSR 0x805 (sha256_f) path, activates the Sha256f air.
- **modexp_precompile**: Contract calls MODEXP precompile (address 0x05) computing 3^5 mod 13. Tests CSR arith256_mod path.
- **multi_tx_3transfers**: 3 ETH transfers to 3 different receivers in one block. Tests multi-transaction block execution, 3 independent ECDSA recoveries, 4 account state updates, multi-instance Main (3 instances).
- **multi_tx_mixed**: Transfer + SSTORE contract call + Transfer in one block. Tests mixed transaction types, contract code loading mid-block, nonce management across transactions, multi-instance Main (4 instances), activates MemAlignReadByte air.

## Public outputs (proof commitments)

Each proof commits the following values as public outputs (verified on-chain):

| Slots | Field | Size | Purpose |
|-------|-------|------|---------|
| 0 | Status | 4B | 0=ok, 1=tx fail |
| 1-2 | Cumulative gas | 8B | Total gas used |
| 3-10 | Block hash | 32B | Computed block hash |
| 11-18 | Post-state root | 32B | State root after execution |
| 19-26 | Transactions root | 32B | Merkle root of transactions |
| 27-34 | Receipts root | 32B | Merkle root of receipts |
| 35 | Transaction count | 4B | Number of transactions |
| 36-43 | Pre-state root | 32B | State root before execution (for chaining) |
| 44-45 | Block number | 8B | Block number |
| 46-47 | Chain ID | 8B | Chain identifier |
| 48-55 | Parent hash | 32B | Previous block hash (for chain linkage) |

The anchoring contract verifies the proof and reads these values to chain blocks:
`assert(pre_state_root == last_anchored_root)` → store `post_state_root`.

### Instance breakdown

The EVM binary uses ~780K steps for runtime initialization (NativeAOT startup). The actual EVM execution adds:
- ETH transfer: ~3.2M steps (mostly ECDSA `ecrecover`)
- SSTORE: ~4.5M steps (ECDSA + storage + gas accounting)
- Patricia root: ~2.6M additional steps (Keccak256 hashing for trie nodes)

Main instances are sized at N=2^22 (4,194,304 rows). Programs exceeding 4M steps use 2 Main instances with Zisk's continuation mechanism.

## Building the EVM binary

The EVM binary is built using the bflat RISC-V compiler (NativeAOT for RISC-V):

```bash
# In Docker (recommended):
docker run --rm -v $(pwd):/work nethereum/bflat-riscv64 \
  bflat build --target:linux-riscv64 --os:none -o /work/evm_raw \
  /work/src/Nethereum.EVM.Zisk/Zisk/*.cs \
  /work/src/Nethereum.EVM.Core/**/*.cs \
  # ... (see zisk/scripts/build.sh for full file list)

# Post-processing:
python3 patch_elf.py --split-code-data --set-tss 0x20000 evm_raw evm_nocookie
# NOP the security cookie write (see ZISK_ISSUES_FOR_TEAM.md #6)
```
